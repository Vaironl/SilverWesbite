User Stories

Website User

A Website User is a consumer that accesses the program via the Website.

Main Paths:
1. A Website user opens the website to research the crime levels in an area.
2. The website requests access to the user’s location.
3. The website loads a map of the user’s current location.
4. The website creates a heat map overlay over the map of crime data in the area displayed.
5. The User identifies areas of concern based on the colors displayed.
6. The User hovers the pointer over areas of interest and receives numeric statistics of the crimes that contributes to the score.

User Stories:
As a Website User I would like to enter the zip code of the location of interest.
As a Website User I would like to see a street map of the area of interest.
As a Website User I would like to see a heat map layer on the street map indicating the safety of an area.
As a Website User, I would like to be able view crime reports and local news to the address I have provided on a separate tab.
As a Website User, I would like to be presented computation statistics when hovering the pointer over an area.
As a Website User, I would like to be notified if the database has no crime data for an area.
As a Website User, I would like to share the current view via a social media platform.
As a Website User, I would like to open a crime map based on a shared link.
As a Website User, I would like to assign weights to each type of crime.
As a Website User, I would like the ability filter out crimes based on crime type.
As a Website User, I would like ability to filter out crimes based on severity.
As a Website User, I would like the ability to filter out crimes based on age.
As a Website User, I would like the ability to filter out crimes based on day of the week.
As a Website User, I would like the ability to filter out crimes based on time of day.

  • Mobile Application User

    A Mobile Application User is a consumer that accesses the program via the Mobile Application.

    Main Paths:
    1. The User starts the application.
    2. The User identifies what danger levels are of concern.
    3. The Users moves about in a safe location while the application checks the safety of an area.
    4. The User moves into an area that is above the danger threshold and receives an alert from the application that the User is in a dangerous area.
    5. The User returns to a safe location and receives an alert that the User is now in a safe location.

    User Stories:
    As an Application User, I would like to enter the zip code of the location of interest.
    As an Application User, I would like to see a street map of the area of interest.
    As an Application User, I want the application to track my location.
    As an Application User, I would like to see a heat map layer on the street map indicating the safety of an area.
    As an Application User, I would like the ability filter out crimes based on crime type.
    As an Application User, I would like ability to filter out crimes based on severity.
    As an Application User, I would like the ability to filter out crimes based on age.
    As an Application User, I would like the ability to filter out crimes based on day of the week.
    As an Application User, I would like the ability to filter out crimes based on time of day.
    As an Application User, I want to be able to identify the levels of danger I am comfortable with.
    As an Application User, I want to be able to assign weights to crimes that are most relevant to me.
    As an Application User, I want the application to track my current safety state.
    As an Application User, I want the application to alert me when my current safety state changes.
    As an Application User, I want to be notified if the application doesn’t have the data needed to track my safety.

    Administrator('s)

    An administrator is a person that provides crime data to be entered in the database.

  • Main Path
    1. The administrator accesses the database webpage.
    2. The administrator signs in.
    3. The administrator selects a .csv file from his or her local machine to submit.
    4. The system validates the .csv file.
    5. The system notifies the administrator that the file is accepted.
    6. The administrator logs off.
    7. The system updates the database with new entries for non-duplicate data.

    User Stories:
    As an Administrator I want to securely log into the website.
    As an Administrator I want to submit crimes data in .csv format.
    As an Administrator I want my submitted data to be validated and the result returned.
    As an Administrator I want the system to automatically ignore line items in the .csv file that would result in duplicate entries.
